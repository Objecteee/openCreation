**日期：** 2026-04-03
**版本：** 1.0
**状态：** 待用户确认

---

## 1. 产品定位与核心价值

**产品名称（暂定）：** 多栖 / ContentBridge

**一句话定位：** 输入一个内容主题，输出多套深度适配不同平台的内容资产。

**核心差异化：** 不只是格式转换，而是深度理解每个平台的用户偏好、内容调性、算法规则，生成"平台原生感"的内容。

**用户痛点：**
- 创作者需要在多平台发布内容，但每个平台的调性、规则、用户喜好差异巨大，手动适配耗时费力
- 现有工具多为"格式适配"（如 Resize 图片），而非"内容调性适配"
- AI 生成内容往往泛泛，缺乏针对特定平台的深度优化

## 2. 目标平台与适配规则

初期覆盖以下平台：

| 平台 | 内容形态 | 核心适配要点 |
|------|---------|-------------|
| 小红书 | 图文笔记 / 短视频 | 标题党 + 干货感 + emoji + 女性化语言 + 热门标签 |
| 抖音 | 短视频脚本 | 3秒开场 + 节奏紧凑 + 悬念钩子 + 行动号召 |
| 微信公众号 | 图文文章 | 深度叙事 + 分段标题 + 引导关注话术 |
| B站 | 视频脚本 | 趣味梗 + 互动感 + 长脚本叙事 + 弹幕友好 |
| 微博 | 短图文 | 热点蹭点 + 简短有力 + 话题标签 |

每个平台的适配规则包含：
- **标题策略**（字数限制、关键词位置、悬念写法）
- **内容结构**（开头钩子、中间叙事、结尾行动号召）
- **语言风格**（正式/轻松/年轻化/术语密度）
- **标签策略**（平台热门标签、垂直标签组合）
- **格式规范**（图片比例、视频时长、文案字数范围）

## 3. 核心功能模块

### 3.1 主题输入层
- 用户输入一个主题或关键词（如"春日穿搭"）
- 可选：补充说明（目标受众、内容方向、风格偏好）
- 支持：文字描述 / 上传参考素材

### 3.2 AI 内容生成引擎
- 基于主题，AI 生成各平台的原始内容
- 每个平台独立生成一套内容（非简单复制改写）
- 支持：大纲生成 → 用户确认 → 完整内容生成（两阶段）

### 3.3 平台深度适配层（核心壁垒）
- 将原始内容按照目标平台的适配规则进行二次加工
- 标题改写、emoji 添加/移除、语言风格调整、标签推荐
- 结构重组（段落顺序、开头钩子、结尾引导）

### 3.4 内容预览与编辑
- 每个平台的内容单独预览
- 支持手动微调（AI 生成 + 人工精修）
- 一键复制 / 直接发布

### 3.5 多平台发布管理
- 账号绑定（各平台 OAuth 授权）
- 内容排期（定时发布）
- 发布状态追踪（曝光、点赞、评论数据回传）
- 数据分析面板（各平台内容表现对比）

### 3.6 积分系统
- 用户注册赠送初始积分
- 每次生成内容包消耗积分
- 充值购买积分
- 积分定价：约 1 元 = 10 积分，每生成一套平台内容包约消耗 5-10 积分

## 4. 技术架构

```
┌─────────────────────────────────────────────────┐
│                   客户端层                        │
│              (React Native App)                  │
│  主题输入 | 预览编辑 | 账号管理 | 数据看板          │
└────────────────────┬────────────────────────────┘
                     │ HTTPS / REST API
┌────────────────────▼────────────────────────────┐
│                  API 网关层                       │
│           (Koa / Express + Nginx)                │
│    鉴权 | 限流 | 路由分发 | 请求日志               │
└──────┬──────────┬──────────┬──────────┬──────────┘
       │          │          │          │
┌──────▼──┐ ┌─────▼───┐ ┌────▼────┐ ┌───▼──────┐
│内容生成  │ │平台适配  │ │用户账号  │ │发布管理   │
│服务      │ │服务      │ │服务      │ │服务       │
│(AI模型) │ │(规则引擎) │ │(数据库)  │ │(平台API) │
└─────────┘ └─────────┘ └─────────┘ └──────────┘
```

**技术栈选型：**

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| 移动端 | React Native + TypeScript | 跨平台，iOS/Android 双端 |
| 状态管理 | Zustand | 轻量，适合 RN 项目 |
| 后端框架 | Node.js + Koa | 快速开发，生态丰富 |
| AI 模型 | GPT-4o / Claude 3.5 | 通过 API 调用，平台适配提示词工程 |
| 数据库 | PostgreSQL + Redis | 主数据 + 缓存 |
| 文件存储 | 阿里云 OSS / AWS S3 | 存储用户上传和生成的内容 |
| 部署 | 阿里云 ECS / Vercel | 后端 + 前端分离部署 |
| CI/CD | GitHub Actions | 自动化构建和发布 |

**平台 API 对接：**
- 抖音开放平台（内容发布 API）
- 小红书开放平台（企业号发布 API）
- 微信公众号 API（草稿箱 + 发布）
- B站开放平台（投稿 API）
- 微博开放平台（发布微博 API）

## 5. 用户使用流程

```
1. 用户打开 App
2. 输入内容主题（如"618 购物节促销"）
3. 选择目标平台（可多选：☑小红书 ☑抖音 ☑公众号）
4. 补充说明（可选）
5. 点击"生成内容"
6. 系统消耗积分，调用 AI 生成
7. 各平台内容分别展示预览
8. 用户可微调或直接使用
9. 绑定平台账号后，一键发布或定时发布
10. 后续可在 App 内查看发布数据
```

## 7. 竞品分析

| 竞品 | 优势 | 不足 |
|------|------|------|
| 牛片网 | 模板丰富，影视脚本能力强 | 主要面向视频，非多平台分发 |
| 剪映"剪同款" | 字节系生态强大 | 主要是视频剪辑，非内容生成 |
| 微信公众号编辑器 | 排版能力强 | 无 AI 生成，非多平台 |
| 秒创 / 腾讯智影 | AI 生成能力 | 主要长视频，多平台适配弱 |

**我们的机会窗口：**
- 目前市场上缺少"AI 生成 + 深度多平台适配"的一体化工具
- 创作者多平台运营需求持续增长
- AI 大模型能力已足够支撑内容生成质量
